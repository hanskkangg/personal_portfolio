How I Deployed My Project to Azure

Step-by-step guide on what I did to deploy my project using Azure CLI and GitHub.

Step 1: Cloning My GitHub Repository

First, I went to the azure terminal and cloned my GitHub repository to my local computer. This creates a local copy of my project so I can make changes and work on it. 

git clone https://github.com/hanskkangg/hansdevhub_static_web_hosting.git

After running this, I had all my project files ready to work on locally.

Step 2: Setting Up My GitHub User Information

Next, I configured Git with my name and email. This ensures that all my commits are properly associated with me. 

git config --global user.email "hans.kkang@gmail.com"
git config --global user.name "hans kang"

This step is important because it lets Git know who is making changes to the project.

Step 3: Preparing My Project for Git

Then, I moved into my project folder and initialized a Git repository. After that, I staged all the files to prepare them for my first commit. 

cd hansdevhub_static_web_hosting
git init
git add .

- cd hansdevhub_static_web_hosting: This command takes me into my project folder.
- git init: This initializes a Git repository in the folder.
- git add .: This stages all the files in the project for a commit.

Step 4: Committing My Changes

After staging my files, I made my first commit with a message describing the changes I made. In this case, I mentioned that I updated my server.js file. 

git commit -m "azure deployment"

This creates a snapshot of the current state of my project, which I can now push to a remote repository or a deployment platform.

Step 5: Configuring Azure Deployment

I set up Azure to accept my local Git repository as a source for deployment. This command provides a URL where I can push my project code. 

az webapp deployment source config-local-git --name name --resource-group group

In this command, I replaced name and resource group with my nme and my resource group

This step is critical because it links my project to Azure Web App for deployment.

Step 6: Retrieving Publishing Credentials

To deploy my project, I needed the publishing username and password provided by Azure. I retrieved these using the following command: 

az webapp deployment list-publishing-credentials --name name --resource-group group

In this command, I replaced name and resource group with my nme and my resource group

Azure returned my username and password, which I used in the next step to add Azure as a remote repository.

Step 7: Adding Azure as a Remote Repository

After getting my credentials, I added Azure Web App as a remote repository in Git. This step lets me push my project code directly to Azure. 

git remote add azure 'https://$hansdevhub:password@hansdevhub.scm.azurewebsites.net/hansdevhub.git'

In this command, I replaced password with the publishing password I retrieved earlier.

Step 8: Pushing My Code to Azure

Finally, I pushed my project code to Azure's master branch using the following command: 

git push azure main:master

This command sends my local project files to Azure, where they are deployed. After the push is complete, my application is live on Azure!
